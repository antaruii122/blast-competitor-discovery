# Gaming Mice Competitive Matching Guide

> **CRITICAL INSTRUCTION:** Before assessing this product, you MUST check `parity_rules.md` to define the strict Dealbreakers (Critical vs Variable specs). The points and rules below are for **qualitative context** only. Do NOT use them to define what is strict vs variable.

## 1. Qualitative Context & "Product Soul"

### Grip Style Optimization (20 points)
**Definition:** The mouse shape optimized for specific grip styles.

**Values:**
- `palm` - Ergonomic shape for palm grip (full hand contact)
- `claw` - Smaller hump for claw grip (arched fingers)
- `fingertip` - Flat/low profile for fingertip grip
- `universal/ambidextrous` - Works for multiple grip styles

**Why Critical:** Ergonomic mice (palm) vs ambidextrous (claw/fingertip) = very different hand comfort.
- Palm gripper using fingertip mouse = hand cramps
- Claw gripper using ergonomic palm mouse = awkward positioning

**Rejection Rule:**
- Extreme ergonomic (right-hand palm) vs ambidextrous = usually different preference
- BUT: Universal mice can compete with specific grip styles

**Tolerance:** Universal ↔ Any style = acceptable. Specific ↔ Specific = must match grip type.

---

### Sensor Tier (20 points)
**Definition:** The optical sensor quality and DPI range.

**Tiers:**
- `budget_<12K_DPI` - Budget sensors (basic gaming)
- `midrange_12K-25K_DPI` - Mainstream gaming sensors
- `flagship_>25K_DPI` - High-end sensors (enthusiast/esports)

**Logic:** 12K DPI vs 25K DPI = both work for most users, but tier signal.
- 6K DPI vs 30K DPI = very different market (office vs esports)

**Rejection Rule:**
- Crossing tiers is acceptable IF same use case
- <5K DPI (office mouse) vs >20K DPI (esports) = **different markets**

**Key Insight:** DPI number is less important than sensor quality (PixArt 3360, 3370, 3389, 3395 are gold standards).

## 2. Structural Features

### Connection Type (20 points)
**Acceptable Range:** Depends on competitiveness level

**Values:**
- `wired` - USB cable (zero latency)
- `wireless_2.4GHz` - Wireless dongle (low latency, gaming-grade)
- `bluetooth` - Bluetooth wireless (office/casual, higher latency)
- `tri-mode` - All three modes (maximum versatility)

**Logic:**
- Competitive gamers prefer wired or 2.4GHz wireless
- Casual gamers and office users accept Bluetooth
- 2.4GHz wireless (1ms) vs Bluetooth (10-20ms latency) = significant

**Tolerance:**
- Wired vs 2.4GHz wireless = acceptable (both gaming-grade)
- 2.4GHz vs Bluetooth = different use cases
- Tri-mode is superior upgrade

---

### Weight Class (10 points)
**Acceptable Range:** Within same class

**Classes:**
- `ultralight_<60g` - FPS/competitive gaming (fast flicks)
- `light_60-80g` - Standard gaming weight
- `standard_>80g` - Heavier mice (MMO, office)

**Logic:** FPS players want ultralight. MMO players accept heavier (more buttons). Office users don't care.

**Tolerance:** ±20g acceptable within class. But 55g vs 110g = very different use cases.

---

## 3. Convenience Features

### Programmable Buttons (10 points)
**Tolerance:** ±2 buttons acceptable

**Classes:**
- `basic_2-4_buttons` - Left, right, middle, maybe 1-2 side (office mice)
- `standard_5-8_buttons` - Gaming mice (thumb buttons)
- `MMO_12+_buttons` - MMO gaming mice (side button grid)

**Logic:** 6 vs 8 buttons = acceptable. But 2 vs 12 = office vs MMO mouse.

---

### RGB Lighting (10 points)
**Definition:** Aesthetic lighting zones.

**Values:**
- `none` - No lighting (office/minimal)
- `single_zone` - Logo or scroll wheel LED
- `multi_zone` - Multiple RGB zones

**Logic:** RGB is aesthetic preference. Not critical.

---

## 4. Secondary Features

### Polling Rate (5 points)
**Tolerance:** ±50% acceptable

**Standard:** 1000Hz (1ms) for gaming
**Enthusiast:** 8000Hz (marketing, minimal real benefit)

**Logic:** 500Hz vs 1000Hz = negligible. 8000Hz = overkill.

---

### Click Durability (5 points)
**Tolerance:** ±30% acceptable

**Standard:** 50M clicks
**Premium:** 80M clicks

**Logic:** Marketing spec, users rarely reach limit.

## 5. Verification Checklist

### Pre-Search
- [ ] Identify grip style (palm/claw/fingertip/universal)
- [ ] Determine sensor tier (budget/<12K, mid/12-25K, flagship/>25K)
- [ ] Note connection preference (wired/2.4GHz wireless/Bluetooth)
- [ ] Check weight class (ultralight/<60g, light/60-80g, standard/>80g)
- [ ] Define use case (FPS competitive, MMO, casual, office)

### During Search
- [ ] Find official product page
- [ ] **Verify grip optimization:**
  - Ergonomic shape (right-hand only) vs ambidextrous?
  - Marketing mentions palm/claw/fingertip?
- [ ] **Check sensor:**
  - DPI range (max DPI)
  - Sensor model (PixArt 3360, 3370, 3389, etc.)
- [ ] **Confirm connection:**
  - Wired only, wireless only, or hybrid?
  - If wireless: 2.4GHz or Bluetooth?
- [ ] **Read specifications:**
  - Weight (grams)
  - Number of programmable buttons
  - Polling rate
  - RGB zones

### Customer Persona Test
**Question:** *"If customer wants Target Mouse, would they consider Candidate?"*

**Consider:**
- Same grip optimization?
- Same use case (FPS vs MMO vs office)?
- Same connection philosophy (competitive vs casual)?
- Same weight preference (ultralight vs standard)?

## 6. Reasoning Examples

**Target: Logitech G Pro Wireless**
- Grip: Ambidextrous (claw/fingertip)
- Sensor: HERO 25K (flagship)
- Connection: 2.4GHz wireless
- Weight: 80g
- Buttons: 8 programmable

**Candidate: Razer Viper Ultimate**
- Grip: Ambidextrous (claw/fingertip)
- Sensor: Focus+ 20K (flagship)
- Connection: 2.4GHz wireless
- Weight: 74g (lighter!)
- Buttons: 8 programmable

**Analysis:**
- **CRITICAL - Grip:** ✅ Both ambidextrous for claw/fingertip
- **CRITICAL - Sensor:** ✅ Both flagship tier (20K vs 25K negligible)
- **STRUCTURAL - Connection:** ✅ Both 2.4GHz wireless (competitive)
- **STRUCTURAL - Weight:** ✅ Both light class (74g vs 80g)
- **CONVENIENCE - Buttons:** ✅ Both 8 buttons

**Match Score:** 98/100

**Why It's a Match:** Perfect competitors. Both are flagship ambidextrous wireless gaming mice for competitive FPS players. Weight and sensor are nearly identical tier. Price is comparable.

## 7. Common Mistakes

### ❌ Mistake 1: "Assuming high DPI = better mouse"
**Why Wrong:** DPI is marketing. Sensor quality (PixArt model) matters more.

**Fix:** Check sensor model, not just max DPI.

---

### ❌ Mistake 2: "Matching ergonomic vs ambidextrous freely"
**Why Wrong:** Hand comfort is personal. Right-handed ergonomic can't be used by lefties.

**Fix:** Verify grip optimization. Universal mice can match both, but specific ergonomic matches specific ergonomic.

---

### ❌ Mistake 3: "Treating Bluetooth vs 2.4GHz as same wireless"
**Why Wrong:** 2.4GHz = 1ms latency (gaming). Bluetooth = 10-20ms (casual).

**Fix:** Verify wireless type. Competitive gamers need 2.4GHz, not Bluetooth.

## 8. Market Tier Indicators

### Budget (<$30)
- Budget sensor (<12K DPI)
- Wired or basic wireless
- Standard ergonomic or ambidextrous
- 5-8 buttons
- Budget brands (Redragon, Razer DeathAdder Essential)

### Mid-Range ($30-80)
- Midrange sensor (12-20K DPI)
- 2.4GHz wireless or wired
- Lightweight (60-90g)
- RGB lighting
- 6-10 buttons
- Premium brands (Logitech G, Razer, SteelSeries)

### Premium (>$80)
- Flagship sensor (PixArt 3370+, 25K+ DPI)
- 2.4GHz wireless + long battery
- Ultralight (<65g) or wireless ultralight
- Per-zone RGB or minimal design
- 5-8 buttons (FPS) or 12+ (MMO)
- Premium brands (Logitech G Pro, Razer Viper/Basilisk Ultimate)
